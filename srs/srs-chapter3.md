# Chapter 3: System Analysis

## 3.1 Fact-Gathering Techniques Used

To ensure the SmartStream system addresses the exact needs of KTD Lanka (Pvt) Ltd., the following fact-finding techniques were employed:

- **Structured Interviews:** Conducted multiple online sessions with the Director (Tharidu Dilan / KT Deshika) to establish baseline requirements, validate pain points, and define Role-Based Access Control (RBAC) matrices[cite: 3, 4].
- **Document Analysis:** Reviewed current manual Excel sheets and paper invoice workflows to identify data duplication issues and tracking bottlenecks[cite: 3, 4].
- **Observation & Requirement Prototyping:** Analyzed the physical workflow of ramen and beverage shipments to design the automated alerting and AI OCR invoice processing pipelines[cite: 2].

## 3.2 Existing System Analysis & Pain Points

KTD Lanka currently utilizes a highly manual, disconnected system relying heavily on Microsoft Excel for store and inventory data, and paper-based invoices for distribution[cite: 2, 4].

**Key Pain Points:**

- **Data Inaccuracy & Duplication:** Excel entries are highly prone to human error, resulting in duplicated or inconsistent records[cite: 2, 4].
- **Delayed Decision Making:** Directors lack real-time visibility into business performance across locations, relying on manual invoice reviews[cite: 2, 4].
- **Reactive Stock Management:** Stock shortages are identified too late, leading directly to lost sales and decreased customer satisfaction[cite: 2, 4].
- **Inefficient Communication:** Manual updates via phone calls and emails between management, inventory staff, and distributors slow down the supply chain[cite: 2, 4].

## 3.4 Use Case Descriptions

### UC-01: View Executive Dashboard

| Field                      | Content                                                                                                                                                                                                                                                                       |
| :------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case ID**            | UC-01                                                                                                                                                                                                                                                                         |
| **Use Case Name**          | View Executive Dashboard                                                                                                                                                                                                                                                      |
| **Actor(s)**               | Director                                                                                                                                                                                                                                                                      |
| **Preconditions**          | User is logged into the system with the 'Director' role assigned via RBAC[cite: 3, 4].                                                                                                                                                                                        |
| **Postconditions Success** | System successfully loads real-time performance reports, sales data, and high-level inventory status[cite: 2].                                                                                                                                                                |
| **Postconditions Failure** | Dashboard fails to load; system displays a "Data Retrieval Error" message.                                                                                                                                                                                                    |
| **Main Flow**              | 1. Director navigates to the "Executive Dashboard" module.<br>2. System authenticates the RBAC permissions.<br>3. System retrieves real-time sales and inventory data from the cloud database.<br>4. System renders interactive charts and performance metrics on the screen. |
| **Alternative Flow 1**     | **At step 2:** If the user's session has expired, the system redirects the Director to the login screen and displays a "Session Expired" message.                                                                                                                             |
| **Alternative Flow 2**     | **At step 3:** If the cloud database is temporarily unreachable, the system loads the most recently cached data and displays a "Showing Offline Cached Data" warning.                                                                                                         |
| **Business Rules**         | **BR-01:** Only the Director role has permission to view aggregate financial performance data[cite: 2].                                                                                                                                                                       |

### UC-02: Manage Stock Levels

| Field                      | Content                                                                                                                                                                                                                                                                                                         |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case ID**            | UC-02                                                                                                                                                                                                                                                                                                           |
| **Use Case Name**          | Manage Stock Levels                                                                                                                                                                                                                                                                                             |
| **Actor(s)**               | Inventory Staff                                                                                                                                                                                                                                                                                                 |
| **Preconditions**          | User is authenticated with the 'Inventory Staff' role[cite: 3, 4].                                                                                                                                                                                                                                              |
| **Postconditions Success** | Product quantities are updated in the database, and the new stock level is visible across the system.                                                                                                                                                                                                           |
| **Postconditions Failure** | Stock is not updated; an error is logged.                                                                                                                                                                                                                                                                       |
| **Main Flow**              | 1. Inventory Staff navigates to the "Inventory Management" module.<br>2. User searches for a specific product (e.g., Korean ramen)[cite: 2].<br>3. User selects "Update Stock" and inputs the new received quantity.<br>4. User clicks "Save".<br>5. System validates the input and updates the cloud database. |
| **Alternative Flow 1**     | **At step 5:** If the user inputs a negative number or invalid character, the system prevents the save and highlights the field with an "Invalid Input" error.                                                                                                                                                  |
| **Alternative Flow 2**     | **At step 5:** If the updated stock level drops below the predefined minimum threshold, the system automatically triggers `UC-04: Generate Low-Stock Alert`.                                                                                                                                                    |
| **Business Rules**         | **BR-02:** All manual stock adjustments must be logged with a timestamp and the specific User ID of the Inventory Staff member.                                                                                                                                                                                 |

### UC-03: Update Distribution Status

| Field                      | Content                                                                                                                                                                                                                                                                                                                                                                     |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case ID**            | UC-03                                                                                                                                                                                                                                                                                                                                                                       |
| **Use Case Name**          | Update Distribution Status                                                                                                                                                                                                                                                                                                                                                  |
| **Actor(s)**               | Distributor                                                                                                                                                                                                                                                                                                                                                                 |
| **Preconditions**          | User is logged in as a 'Distributor' and has active assigned shipments[cite: 2].                                                                                                                                                                                                                                                                                            |
| **Postconditions Success** | The delivery status is updated, and management/inventory staff can see the real-time update.                                                                                                                                                                                                                                                                                |
| **Postconditions Failure** | Status remains unchanged; error message is shown to the Distributor.                                                                                                                                                                                                                                                                                                        |
| **Main Flow**              | 1. Distributor opens the "Supply Chain Updates" module.<br>2. System lists all active dispatches assigned to the Distributor.<br>3. Distributor selects a specific dispatch and changes the status (e.g., "In Transit" to "Delivered").<br>4. Distributor adds an optional text note and clicks "Confirm Update".<br>5. System updates the shipment record in the database. |
| **Alternative Flow 1**     | **At step 2:** If the Distributor has no active shipments, the system displays "No Active Dispatches" and hides the status update form.                                                                                                                                                                                                                                     |
| **Alternative Flow 2**     | **At step 4:** If the Distributor attempts to change the status to "Delivered" but the GPS/location data does not match the retail destination, the system prompts for a manual override reason.                                                                                                                                                                            |
| **Business Rules**         | **BR-03:** Distributors can only view and update data for shipments explicitly assigned to them.                                                                                                                                                                                                                                                                            |

### UC-04: Generate Low-Stock Alert

| Field                      | Content                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case ID**            | UC-04                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Use Case Name**          | Generate Low-Stock Alert                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Actor(s)**               | System                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Preconditions**          | The database registers that a product's stock quantity has dropped below its configured minimum threshold.                                                                                                                                                                                                                                                                                                                                   |
| **Postconditions Success** | Email and SMS notifications are successfully dispatched to relevant Inventory Staff and Management[cite: 2, 4].                                                                                                                                                                                                                                                                                                                              |
| **Postconditions Failure** | The alert fails to send; the failure is logged in the system error queue for retry.                                                                                                                                                                                                                                                                                                                                                          |
| **Main Flow**              | 1. System detects a low-stock trigger event during a database update.<br>2. System identifies the specific product and its current quantity.<br>3. System queries the RBAC database to retrieve contact details (Email/Phone) of the assigned Inventory Staff[cite: 3].<br>4. System formats the alert message.<br>5. System pushes the message to the Email and SMS API gateways.<br>6. System logs the alert transmission in the database. |
| **Alternative Flow 1**     | **At step 5:** If the primary SMS API gateway is unresponsive, the system automatically attempts to send the notification via a fallback secondary gateway.                                                                                                                                                                                                                                                                                  |
| **Alternative Flow 2**     | **At step 3:** If no specific Inventory Staff member is assigned, the system routes the alert to the primary Director account.                                                                                                                                                                                                                                                                                                               |
| **Business Rules**         | **BR-04:** Alerts must contain the exact product name, current quantity, and the time the shortage was detected.                                                                                                                                                                                                                                                                                                                             |

### UC-05: Trigger Automated Cloud Backup

| Field                      | Content                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| :------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case ID**            | UC-05                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Use Case Name**          | Trigger Automated Cloud Backup                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Actor(s)**               | System                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Preconditions**          | The system clock reaches the scheduled backup interval (e.g., 12:00 AM daily).                                                                                                                                                                                                                                                                                                                                                                                              |
| **Postconditions Success** | All database records, inventory data, and secured documents are successfully encrypted and copied to the secure cloud storage layer[cite: 2, 4].                                                                                                                                                                                                                                                                                                                            |
| **Postconditions Failure** | Backup fails; an urgent notification is sent to the IT administration team and the Director.                                                                                                                                                                                                                                                                                                                                                                                |
| **Main Flow**              | 1. System initiates the automated backup protocol at the scheduled time.<br>2. System compiles and compresses the current database state.<br>3. System encrypts the compiled data payload.<br>4. System establishes a secure connection to the designated disaster recovery cloud server[cite: 3].<br>5. System uploads the encrypted payload.<br>6. Cloud server confirms successful receipt and integrity of the file.<br>7. System logs the successful backup timestamp. |
| **Alternative Flow 1**     | **At step 4:** If the connection to the primary cloud server fails, the system automatically attempts to connect to a secondary/failover cloud server region.                                                                                                                                                                                                                                                                                                               |
| **Alternative Flow 2**     | **At step 6:** If the cloud server reports data corruption during the upload, the system aborts the transfer, deletes the partial file, and restarts the upload.                                                                                                                                                                                                                                                                                                            |
| **Business Rules**         | **BR-05:** Backup files must be retained in the cloud for a minimum of 30 days before being overwritten or archived.                                                                                                                                                                                                                                                                                                                                                        |
