# Chapter 1: Introduction

## 1.1 Purpose and Scope

The purpose of this Software Requirements Specification (SRS) is to formally define the system requirements for **SmartStream**, an Integrated Business Management System developed for **KTD Lanka (Pvt) Ltd**.

The scope of SmartStream encompasses the unification of inventory, accounting, and executive analytics into a single, secure cloud-based platform. The system is designed to replace the client's current reliance on disconnected Microsoft Excel spreadsheets and manual invoicing processes. Core capabilities include smart inventory management with automated Email/SMS low-stock alerts, a remote executive dashboard for real-time performance monitoring, role-based access control (RBAC), and automated cloud disaster recovery to ensure data security.

## 1.2 Definitions, Acronyms, Abbreviations

- **SRS:** Software Requirements Specification.
- **SDS:** Software Design Specification.
- **RBAC:** Role-Based Access Control – A method of restricting network access based on the roles of individual users within an enterprise.
- **Disaster Recovery (DR):** The process of restoring data and system functionality after a disruptive event, facilitated in this project via cloud-based storage.
- **COTS:** Commercial Off-The-Shelf (referring to existing market alternatives like Zoho or QuickBooks).
